RDP远程登录Linux

时间： 2017-05-27   |   分类： tech     |   阅读： 237 字 ~1分钟

有时候为了远程调试，经常会遇到RDP到Linux主机的情况。

主要用到xrdp协议并确保Linux主机已安装桌面系统。

连接到Ubuntu

sudo apt-get install -y xrdp xfce4
echo  xfce4-session >~/.xsession

sudo vi  /etc/xrdp/startwm.sh

之后编辑内容如下：

#!/bin/sh

if [ -r /etc/default/locale ]; then
  . /etc/default/locale
  export LANG LANGUAGE
fi
# 注释掉下面一行，如果RDP停在登录界面
. /etc/X11/Xsession
startxfce4

之后重启服务：

sudo service xrdp restart

对于命令行tab失效问题的解决： Go to “Applications -> Settings->Window Manager”, in the “Keyboard” tab, clear the binding of “Switch wndow for same application -> Tab”

连接到CentOS7

# 安装桌面系统
yum groupinstall "GNOME Desktop" "Graphical Administration Tools"

rpm -Uvh https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm
yum -y install xrdp tigervnc-server

# 启动xrdp
systemctl start xrdp
# netstat -antup | grep xrdp
# 应该能看到启动的服务
systemctl enable xrdp
